## Seção 2: Especificações de E-mail

### Limites de Linha de Assunto por Cliente

| Cliente de E-mail | Caracteres Visíveis (Desktop) | Caracteres Visíveis (Mobile) |
|---|---|---|
| Gmail | 70 | 40 |
| Outlook (desktop app) | 73 | 38 |
| Outlook.com | 60 | 38 |
| Apple Mail | 78 | 35 |
| Yahoo Mail | 46 | 35 |
| Samsung Mail | N/A | 33 |

**Recomendação:** Mantenha as linhas de assunto com menos de 40 caracteres para renderização mobile confiável entre clientes. Coloque as palavras mais importantes no início.

### Texto de Preview (Preheader)

| Cliente de E-mail | Comprimento Visível do Texto de Preview |
|---|---|
| Gmail | 90-110 characters (varies by subject line length) |
| Outlook | 35-90 characters |
| Apple Mail | 75-100 characters |
| Yahoo Mail | 55-70 characters |

**Recomendação:** Escreva 40-90 caracteres de texto de preview. Se não definido explicitamente, os clientes de e-mail puxam o primeiro texto visível do corpo. Use texto de preheader oculto no HTML para controlar isso.

### Specs do Corpo do E-mail

| Especificação | Valor |
|---|---|
| Largura recomendada do corpo | 600 px (max 640 px) |
| Limite de corte do Gmail | 102 KB (total HTML size including inline CSS). Messages exceeding this are truncated with a "[Message clipped] View entire message" link. |
| Tamanho máx. do e-mail (prático) | Keep under 100 KB HTML. Total with images should stay under 1 MB for fast loading. |
| Pilhas de fontes | System fonts for reliability: Arial, Helvetica, Georgia, Times New Roman. Web fonts supported in Apple Mail, iOS Mail, Android (default), Outlook.com, Thunderbird. NOT supported in Gmail, Outlook desktop. |
| Tamanho mínimo de fonte (mobile) | 14 px body, 22 px headlines (iOS auto-sizes text below 13 px) |
| Altura da linha | 1.4-1.6 for body text |
| Imagens de fundo | Supported in most clients; NOT supported in Outlook desktop (use VML fallback) |

### Tratamento de Imagens

| Especificação | Valor |
|---|---|
| Formatos de imagem | JPEG, PNG, GIF. WebP supported in Gmail, Apple Mail; NOT in Outlook. |
| Suporte retina | Use 2x resolution images displayed at 1x size (e.g., 1200 px image displayed at 600 px width) |
| Bloqueio de imagem | Outlook, some corporate clients block images by default. Always include alt text. |
| GIF animado | Supported everywhere except Outlook desktop (shows first frame only) |
| Largura máx. de imagem única | 600 px display width (1200 px actual for retina) |
| Proporção imagem/texto | Aim for 60% text / 40% images to avoid spam filters |

### Specs do Botão CTA

| Especificação | Recomendação |
|---|---|
| Tamanho mínimo do botão | 44 x 44 px (Apple HIG tap target) |
| Tamanho recomendado do botão | 48-60 px height, full-width on mobile |
| Método do botão | Bulletproof buttons using HTML/CSS (padding-based). VML fallback for Outlook. Avoid image-only buttons. |
| Texto do botão | 2-5 words, action-oriented. Max 30 characters. |
| Contraste do botão | WCAG AA minimum (4.5:1 contrast ratio for text on button color) |

### Considerações de Modo Escuro

| Cliente | Comportamento no Modo Escuro |
|---|---|
| Apple Mail / iOS Mail | Full color inversion with `@media (prefers-color-scheme: dark)` support |
| Gmail (Android) | Partial: inverts light backgrounds to dark, adjusts text to white. Does NOT respect `prefers-color-scheme`. |
| Gmail (iOS) | Partial: similar to Android. Does NOT respect `prefers-color-scheme`. |
| Outlook (desktop) | Full inversion. Some transparent PNGs get white backgrounds. |
| Outlook.com | Partial inversion. Supports `[data-ogsc]` and `[data-ogsb]` selectors. |
| Yahoo Mail | Applies its own dark theme. Limited CSS override support. |

**Regras de design para modo escuro:**
- Adicione padding transparente ao redor de logos (evita conflitos estranhos de cor)
- Use PNGs semitransparentes com bordas contrastantes
- Teste com fundos escuros e claros
- Defina esquemas de cores claro e escuro no CSS onde suportado
- Evite fundos branco puro (#FFFFFF); use #FAFAFA para que a detecção de modo escuro dispare de forma consistente

### Regras de Renderização Mobile

| Regra | Detalhe |
|---|---|
| Abordagem responsiva | Use fluid tables with `max-width: 600px` and `width: 100%` |
| Media queries | Supported in Apple Mail, iOS Mail, Android (default), Thunderbird. NOT supported in Gmail (any), Yahoo Mail, Outlook. |
| Empilhamento de colunas | Use `display: block` on table cells for mobile stacking via media queries, or use a hybrid/fluid approach for clients without media query support |
| Alvos de toque | Minimum 44 x 44 px for all tappable elements |
| Layout em coluna única | Recommended for mobile-first design |

### Apple Mail Privacy Protection (MPP)

| Impacto | Detalhe |
|---|---|
| Rastreamento de abertura | Unreliable. Apple pre-fetches all images (including tracking pixels) via proxy, inflating open rates. Approximately 50-60% of Apple Mail users have MPP enabled. |
| Geolocalização baseada em IP | No longer accurate for Apple Mail users. Apple proxies mask true IP. |
| Detecção de dispositivo | Unreliable. User-agent data is obscured. |
| Resposta recomendada | Shift KPIs to click-through rate, conversion rate, and revenue attribution. Use UTM parameters for tracking. Build engagement segments based on clicks, not opens. |

## Seção 4: Referência de Schema Markup

Use dados estruturados de schema.org (formato JSON-LD recomendado) para melhorar a aparência nos resultados de busca e habilitar rich results.

| Tipo de Schema | Quando Usar | Rich Result |
|---|---|---|
| `Article` | Blog posts, news articles, editorial content. Use `NewsArticle` for timely news, `BlogPosting` for blogs. | Article carousel, headline in Top Stories |
| `FAQPage` | Pages with a list of questions and answers. Must show Q&A on the visible page (not just in markup). | Expandable FAQ dropdowns in SERP |
| `HowTo` | Step-by-step instructional content (recipes, DIY, tutorials). Each step must be a distinct action. | Step-by-step rich result with images/video |
| `Product` | Product pages. Include `name`, `image`, `description`, `offers` (price, availability, currency). | Product snippet with price, availability, rating |
| `LocalBusiness` | Businesses with a physical location. Include address, phone, hours, geo coordinates. Subtype to specific business type (e.g., `Restaurant`, `Dentist`). | Knowledge panel, Maps integration |
| `Organization` | Company/brand homepage. Include name, logo, URL, social profiles, contact info. | Knowledge panel, logo in search |
| `Person` | Author pages, speaker bios, team pages. Link to social profiles and authored content. | Knowledge panel for notable people |
| `Review` / `AggregateRating` | Product reviews, service reviews. `AggregateRating` for summary of multiple reviews. Must represent genuine user reviews. | Star ratings in SERP |
| `Event` | Upcoming events with date, location, ticket info. Include `startDate`, `location`, `offers` for tickets. | Event listing with date, venue, ticket link |
| `VideoObject` | Video content on pages. Include `name`, `description`, `thumbnailUrl`, `uploadDate`, `duration`, `contentUrl` or `embedUrl`. | Video carousel, key moments |
| `Speakable` | Content optimized for text-to-speech / voice assistants. Identify which sections are most suitable for audio playback. | Voice assistant eligibility (Google Assistant) |
| `BreadcrumbList` | All pages with breadcrumb navigation. Reflects the page hierarchy. | Breadcrumb trail in SERP instead of raw URL |
| `SiteNavigationElement` | Main site navigation menus. Helps search engines understand site structure. | Sitelinks refinement (indirect) |

### Notas de Implementação de Schema

- **Formato:** Sempre use JSON-LD (recomendado pelo Google) em vez de Microdata ou RDFa.
- **Posicionamento:** Insira `<script type="application/ld+json">` no `<head>` ou `<body>` da página.
- **Validação:** Teste com o Google Rich Results Test (https://search.google.com/test/rich-results) e o Schema Markup Validator (https://validator.schema.org).
- **Aninhamento:** Aninhe schemas relacionados (ex: `Product` contendo `AggregateRating` e `Offers`).
- **Evite:** Não marque conteúdo que não está visível na página. Não use schema para fins enganosos. O Google pode aplicar ações manuais para dados estruturados enganosos.

## Seção 5: Guia de Formatos de Imagem

### Comparação de Formatos

| Propriedade | WebP | AVIF | PNG | JPEG |
|---|---|---|---|---|
| Tipo de compressão | Lossy + Lossless | Lossy + Lossless | Lossless | Lossy |
| Transparência | Yes | Yes | Yes | No |
| Animação | Yes | Yes (limited tooling) | Yes (APNG) | No |
| Tamanho típico vs JPEG | 25-35% smaller | 40-50% smaller | 5-10x larger | Baseline |
| Velocidade de codificação | Fast | Slow (CPU-intensive) | Fast | Fast |
| Velocidade de decodificação | Fast | Moderate | Fast | Fast |
| Profundidade de cor | 8-bit | 8, 10, 12-bit (HDR support) | 8, 16-bit | 8-bit |
| Dimensões máximas | 16,383 x 16,383 px | No practical limit | No practical limit | 65,535 x 65,535 px |
| Carregamento progressivo | No | Yes | No (interlaced PNG exists) | Yes |

### Suporte de Navegadores (em 2026)

| Format | Chrome | Firefox | Safari | Edge | iOS Safari | Android |
|---|---|---|---|---|---|---|
| WebP | 32+ | 65+ | 16+ | 18+ | 16+ | 4.2+ |
| AVIF | 85+ | 93+ | 16.4+ | 85+ | 16.4+ | 85+ |
| PNG | All | All | All | All | All | All |
| JPEG | All | All | All | All | All | All |

### Quando Usar Cada Formato

| Formato | Ideal Para | Evitar Quando |
|---|---|---|
| **WebP** | General web images. Default choice for photos, illustrations, thumbnails. Best balance of compression, quality, and compatibility. | Email campaigns (limited client support), environments requiring lossless at maximum quality. |
| **AVIF** | Hero images, high-quality photography where file size is critical. Best compression ratio available. Use when encoding time is not a constraint. | Bulk image processing pipelines (slow encoding), older browser support required without fallback, email. |
| **PNG** | Logos, icons, screenshots, images with text, transparency over complex backgrounds, any image requiring pixel-perfect lossless quality. | Photographs (file size too large), any large image where lossy compression is acceptable. |
| **JPEG** | Universal fallback. Email images. Social media uploads (platforms re-encode anyway). Legacy system compatibility. | Images with text (compression artifacts), transparency needed, logos or sharp-edged graphics. |

### Melhor Prática de Implementação

Use o elemento `<picture>` com fallback de formato:

```html
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Description" width="800" height="600" loading="lazy">
</picture>
```

**Regras principais:**
- Sempre inclua atributos `width` e `height` para evitar Cumulative Layout Shift (CLS)
- Use `loading="lazy"` para imagens abaixo da dobra
- Use `loading="eager"` ou `fetchpriority="high"` para imagens LCP (Largest Contentful Paint)
- Sirva tamanhos responsivos com atributos `srcset` e `sizes` para troca de resolução
- Comprima JPEG com qualidade 75-85 para web, WebP com qualidade 75-80, AVIF com qualidade 60-70
- Sempre remova metadados EXIF para entrega web (privacidade e tamanho de arquivo)
